    Suivi de la campagne de test fermé – Yin Shi

    Ce fichier sert à suivre, au fil du temps, les éléments que Google voit dans la Play Console pendant la campagne de test fermé (nouveau compte perso).

    Mise à jour manuelle après chaque revue de la console (1–2 fois par semaine suffit).

    Tableau de suivi

    Date consultation Carte « Production » – état cond. 3 Release test fermée active Liste testeurs (nom → # utilisateurs) Observations Vitals / Acquisition Actions / décisions immédiates
    2025-11-27 Exige 14 jours supplémentaires (depuis 19/11) 1.0.22 (release fermée, sortie 24 nov. 2025) test Béta → 43 Vitals sans avertissement, UAJ/UAM ~9,6 %, pertes ~0,25 %. Données souvent manquantes (volume faible) Relancer/entretenir les testeurs, encourager l’usage réel et le feedback.

    Mode d’emploi rapide

    • Carte « Production » – état cond. 3
      Reporter le texte de la carte sur le tableau de bord (ex. « Votre appli nécessite plus de tests… », « Conditions remplies », etc.).

    • Release test fermée active
      Numéro + date de la release visible dans Tester et publier → Tests → Tests fermés → Alpha.

    • Liste testeurs
      Nom et nombre d’utilisateurs dans l’onglet Testeurs du canal fermé (ex. test Béta → 43).

    • Observations Vitals / Acquisition
      Quelques mots sur :

    • Android Vitals (avertissement ou non, tendance UAJ/UAM, taux de perte),

    • Accroître le nombre d’utilisateurs (acquisitions d’appareils, premières ouvertures, etc.).

    • Actions / décisions immédiates
      Ce que tu décides suite à cette revue (relance testeurs, planifier une nouvelle release, préparer une partie du formulaire « Apply for production », etc.).
