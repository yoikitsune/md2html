    Système complet de gestion des bugs

    Date de création : 11 décembre 2025
    Dernière modification le : 11 décembre 2025
    Dernière modification par : Vanessa LEROY

    Objectif

    L'objectif de ce système est de permettre aux utilisateurs de signaler facilement les bugs rencontrés dans l'application, de suivre leur résolution et de centraliser toutes les informations relatives aux problèmes techniques.

    Stockage

    Les rapports de bugs sont stockés dans la collection Firestore bug_reports avec les champs suivants : - zoneBug: La zone de l'application où le bug a été rencontré - descriptionBug: La description détaillée du bug - email: L'email de l'utilisateur (optionnel) - uid: L'ID utilisateur (si connecté) - platform: La plateforme (Android/iOS) - device_model: Le modèle de l'appareil - device_version: La version du système d'exploitation - app_version: La version de l'application - status: Le statut du bug (open, in_progress, resolved) - createdAt: La date de création - updatedAt: La date de mise à jour

    Signalement

    Les utilisateurs peuvent signaler un bug depuis : - l'écran de paramètres de l'application - l'écran d'inscription / de connexion - l'écran d'accueil

    Le formulaire collecte les informations suivantes : - L'email de l'utilisateur (optionnel pour le suivi) - La zone de l'application où le bug a été rencontré - Une description détaillée du bug

    Le rapport est automatiquement enregistré dans Firebase lors de lveua soumission.

    Les administrateurs ont un écran de gestion des bugs disponible dans leur onglet "Outils" permettant de : - modifier le statut - contacter l'utilisateur - supprimer un rapport de bug - actualiser la liste manuellement

    Suivi des bugs

    Les administrateurs peuvent suivre l'évolution des bugs via leur statut : - open : Le bug est signalé et non traité - in_progress : Le bug est en cours de résolution - resolved : Le bug est résolu

    L'utilisateur n'a pas d'écran pour visualiser le suivi de son signalement.

    Fonctionnement (pour les DEVS)

    Ce système utilise Firebase Firestore pour stocker les rapports de bugs. Les administrateurs peuvent accéder à une interface dédiée pour gérer les signalements, modifier les statuts et contacter les utilisateurs via l'email fourni.

    Les données sont structurées de la manière suivante : - Collection : bug_reports - Document ID : UUID généré automatiquement - Champs : tous les champs définis dans la section "Stockage"

    La gestion des données est réalisée via le processus suivant : - lib/mappers/bug_report_mapper.dart : représentation d'un document ID avec interface Firebase - lib/models/bug_report.dart : modèle métier pour les rapports de bugs - lib/services/bug_report_service.dart : service pour gérer les opérations CRUD sur les rapports de bugs - lib/providers/bug_report_provider.dart : provider pour gérer l'état des rapports de bugs. C'est le point d'entrée et d'interface entre les écrans et la base de données

    Les interfaces utilisateurs sont : - lib/screens/admin_bug_reports_screen.dart : écran d'administration des rapports de bugs - lib/screens/bug_report_screen.dart : formulaire de signalement de bugs pour les utilisateurs - lib/widgets/button/bug_report_button.dart : bouton réutilisable pour signaler un bug (afin de garder un visuel commun)

    Tests à réaliser

    • Suivre le processus de signalement d'un bug jusqu'au bout avec un compte utilisateur
    • Suivre le processus de signalement d'un bug jusqu'au bout sans compte utilisateur
    • En tant qu'Administrateur :
      • modifier le statut
      • contacter l'utilisateur via l'email fourni
      • supprimer un rapport de bug
      • actualiser la liste manuellement
