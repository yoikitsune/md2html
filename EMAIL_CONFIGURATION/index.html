    📧 Configuration de l'envoi d'emails avec Firebase

    1. Configuration DNS pour la délivrabilité

    Pour améliorer la délivrabilité des emails (notamment pour les fournisseurs comme Free.fr), nous avons configuré un domaine personnalisé pour l'envoi d'emails via Firebase Authentication.

    Enregistrements DNS ajoutés (DreamHost) :

    # SPF Record (TXT)
Type: TXT
Host: @
Value: v=spf1 include:_spf.firebasemail.com ~all

# Firebase Verification (TXT)
Type: TXT
Host: @
Value: firebase=yin-shi-1967c

# DKIM Records (CNAME)
Type: CNAME
Host: firebase1._domainkey
Value: mail-yinshi-app.dkim1._domainkey.firebasemail.com

Type: CNAME
Host: firebase2._domainkey
Value: mail-yinshi-app.dkim2._domainkey.firebasemail.com

    2. Configuration Firebase

    Dans la console Firebase :

    1. Aller dans Authentication → Modèles
    2. Configurer le domaine d'envoi personnalisé : yinshi.app
    3. Les emails seront envoyés depuis : noreply@yinshi.app

    3. Délai de propagation

    • DNS : Jusqu'à 48h pour la propagation complète
    • Firebase : Jusqu'à 24h pour la prise en compte complète des paramètres

    4. Vérification

    Pour vérifier que tout est correctement configuré :

    1. Utiliser un outil comme MXToolbox
    2. Vérifier les enregistrements SPF et DKIM
    3. Tester l'envoi d'un email de vérification

    5. Dépannage

    Problèmes courants :

    • Emails non reçus : Vérifier les dossiers spam
    • Erreurs d'envoi : Vérifier les logs Firebase
    • Problèmes de délivrabilité : Vérifier la configuration SPF/DKIM
